# Техническое задание
## Автоматизированная система «Регистрация посетителей» (АС РП)

## 1. Общие сведения
Наименование системы: АС РП.

Назначение: автоматизация учёта посетителей организации.

Основание для разработки: учебный проект по дисциплине, оформление в упрощённом виде по ГОСТ 34.602-89.

## 2. Назначение и цели создания системы
Система предназначена для:
- регистрации посетителей при входе;
- фиксации выхода посетителей;
- просмотра текущих посетителей;
- поиска информации о посетителях;
- формирования простого отчёта в CSV.

Цели создания:
- сократить время регистрации;
- упростить контроль входа и выхода;
- обеспечить хранение данных в базе данных;
- представить наглядный MVP, пригодный для защиты в колледже.

## 3. Характеристика объекта автоматизации
Объект автоматизации: рабочее место оператора регистрации посетителей.

Пользователи системы:
- Operator — регистрирует вход и выход;
- Admin — просматривает данные и формирует отчёт.

## 4. Требования к системе
### 4.1 Требования к функциям
Система должна обеспечивать:
- ввод ФИО посетителя;
- ввод номера документа;
- выбор принимающего сотрудника;
- ввод цели визита;
- автоматическую фиксацию времени входа;
- регистрацию времени выхода по кнопке `Выход`;
- отображение списка текущих посетителей;
- поиск по ФИО или документу;
- выгрузку списка визитов в CSV;
- формирование уникального номера пропуска.

### 4.2 Требования к видам обеспечения
Информационное обеспечение:
- таблица `Visitors`;
- таблица `Visits`;
- таблица `Employees`.

Программное обеспечение:
- backend на ASP.NET Core Web API;
- frontend на Vue 3;
- база данных SQLite.

### 4.3 Требования к надёжности
Для учебного MVP система должна корректно обрабатывать ошибки ввода и основные серверные ошибки.

### 4.4 Требования к интерфейсу
Интерфейс должен быть простым, понятным и пригодным для демонстрации за 5–10 минут.

## 5. Состав и содержание работ
- проектирование структуры БД;
- разработка REST API;
- разработка пользовательского интерфейса;
- тестовая демонстрация регистрации входа, выхода, поиска и выгрузки.

## 6. Порядок контроля и приёмки
Критерии приёмки:
- проект запускается локально;
- регистрация посетителя работает;
- выход фиксируется;
- текущие посетители отображаются;
- поиск работает;
- CSV-отчёт формируется.
